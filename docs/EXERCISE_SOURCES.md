# Exercise bank — sources and inspiration

Exercises in `backend/app/seed_curriculum.py` are **original write-ups** for Automat Washing, aligned to a **15-day** arc. They are **informed by** these public, high-signal resources (cite in mentor reviews when students want depth):

| Topic | Inspiration / further reading |
| --- | --- |
| Linux & shell | Coreutils docs, DigitalOcean process tutorials, general triage drills |
| Git & PR hygiene | `git-scm.com` branching chapters, Conventional Commits, Atlassian merge vs rebase |
| React components & async UI | [React.dev — Thinking in React](https://react.dev/learn/thinking-in-react), MDN `fetch`, WCAG quick reference |
| Forms & validation | [Hands-on React — Lab 16 (Form Validation, TS)](https://handsonreact.com/docs/labs/ts/FormValidation) |
| FastAPI | [FastAPI Tutorial](https://fastapi.tiangolo.com/tutorial/), [Bigger applications](https://fastapi.tiangolo.com/tutorial/bigger-applications/) |
| Testing (Day 10) | [FastAPI — Testing](https://fastapi.tiangolo.com/tutorial/testing/), [pytest docs](https://docs.pytest.org/), [Real Python — pytest](https://realpython.com/pytest-python-testing/), public drill repos (GitHub search: `pytest exercises`, `pytest-fastapi`, ChristopherGS **ultimate-fastapi-tutorial**), [CodeRefinery — testing](https://coderefinery.github.io/testing/) |
| Big-O / pagination (Day 11) | [MDN — Big O notation](https://developer.mozilla.org/en-US/docs/Glossary/Big_O_notation), Big O & time-complexity tutorial (see `PRIMARY_VIDEOS[11]` in `seed_curriculum.py`), common interview list-endpoint prompts |
| Docker | [Docker labs](https://github.com/docker/labs), [Play with Docker classroom](https://training.play-with-docker.com/) |
| Kubernetes | [kubernetes.io concepts](https://kubernetes.io/docs/concepts/), [Kubernetes Course Labs](https://kubernetes.courselabs.co/) |
| CI/CD | [GitHub Actions docs](https://docs.github.com/en/actions) |
| Rancher | [Rancher Manager docs](https://ranchermanager.docs.rancher.com/) |
| System design (URL shortener) | [System Design Handbook — URL shortening](https://www.systemdesignhandbook.com/guides/design-a-url-shortening-service/), [System Design School](https://systemdesignschool.io/problems/url-shortener/solution) |

Each seeded `Exercise` row stores `source_attribution` for mentor-visible provenance inside lesson pages.
